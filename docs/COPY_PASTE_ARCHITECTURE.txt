SMYST.COM FREE-ONLY ARCHITECTURE

Production rule:
- GitHub.com Free for repository, documentation, CI/CD and deployment pipelines.
- Cloudflare.com Free for DNS, TLS, CDN, Pages, Workers, KV, caching, WAF basics and edge delivery.
- IDrive e2 as central S3-compatible storage for files, media, documents, uploads, backups and twin data.
- No paid add-on service is a production requirement.

Phase 1:
- Free-Only MVP.
- Vite/React frontend.
- PWA and Capacitor iOS/Android shells.
- Cloudflare Pages for static delivery.
- Cloudflare Workers for API, auth, upload signing, storage, translation and chat MVP.
- Cloudflare KV for sessions, OAuth/passkey state, quotas, small metadata and upload status.
- IDrive e2 for all large and durable user-owned objects.

Target diagram:

Client Web/PWA/iOS/Android
  -> Cloudflare Pages
  -> Cloudflare Workers
  -> Cloudflare KV for small state
  -> IDrive e2 for files and backups

Data placement:
- Code and docs: GitHub.
- Web artifact: Cloudflare Pages.
- API/auth/storage/chat logic: Cloudflare Workers.
- Sessions, roles, quotas, upload intents, public-safe snapshots: Cloudflare KV.
- Images, video, audio, documents, profile images, backups, twin context objects: IDrive e2.

Security:
- No secrets in the repository.
- No permanent storage keys in clients.
- Signed uploads only.
- Private by default.
- Public profiles are filtered snapshots.
- Private profiles are not indexed.
- Rate limits, CORS, CSRF, CSP, input validation and upload validation are mandatory.

SEO/AEO/GEO:
- robots.txt, sitemap.xml, llms.txt, schema.org, OpenGraph, Twitter Cards and static multilingual landing pages.
- Public profile pages only when visibility and consent allow indexing.

Scale statement:
- Billion-user/day scale is the long-term vision.
- The free-only stack is not a billion-user guarantee.
- Phase 1 focuses on clean architecture, security, fast UX and future portability.
